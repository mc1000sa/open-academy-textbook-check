# 🚨 AI에게: 새 프로젝트 시작 전 반드시 이 파일을 먼저 읽어라

이 파일은 기존 채팅에서 논의된 내용을 이어받아 새 통합 포털 프로젝트를 시작하기 위한 컨텍스트다.
새 채팅창에서 이 파일이 첫 메시지로 제공될 것이므로, 아래 내용을 먼저 숙지하고 작업에 임하라.

---

## 📌 이 프로젝트가 무엇인지

이 폴더는 **세 개의 독립 프로젝트를 하나의 폴더로 복사해서 합친 것**이다.
목표는 세 프로젝트를 **통합 포털(단일 링크)**로 합치는 것이다.

### 합쳐진 세 프로젝트

| # | 프로젝트 | 역할 | 개발 상태 | 원래 Firebase 프로젝트 |
|---|---|---|---|---|
| 1 | **교재 점검 시스템 (OATIS)** | 강사가 교재 점검 입력, 학부모/학생이 결과 확인 | ✅ 안정 운영 중 | 별도 Firebase 콘솔 |
| 2 | **OMR 답안 입력 시스템** | 학생이 수학 시험 답안을 직접 입력 | ✅ 안정 운영 중 (수학과 전체 공유) | 별도 Firebase 콘솔 |
| 3 | **수학자 탐험관** | 학생이 자기 정보 입력 후 수학자 정보를 조사하는 학습 콘텐츠 | 🚧 개발 중단 (추후 재개 예정) | 별도 Firebase 콘솔 (확인 필요) |

---

## 🎯 통합의 목적과 범위

### 왜 합치는가
- **OMR 시스템**은 수학과 전체 강사 공유로 운영됨 → **그대로 독립 유지**
- **교재 점검 시스템**은 수최쌤 담당반만 사용 → **이 프로젝트의 핵심**
- **수학자 탐험관**은 수최쌤 담당반 학생 대상 학습 콘텐츠 → 포털에 메뉴로 통합
- 수최쌤 담당반 학생들은 시험 후 OMR 입력 → 교재 점검 확인을 **사이트를 오가며** 해야 하는 불편이 있었음
- **하나의 링크**로 들어와서 세 가지 기능을 선택할 수 있게 만들어 학생 편의를 높이는 것이 목표

### 사용 구조 (확정)
```
[단일 포털 링크]
        ↓
  로그인/학생 선택
        ↓
 ┌──────────┬──────────┬──────────┐
 │OMR 답안  │ 교재점검  │수학자 탐험관│
 │  입력    │   확인   │ (재개 예정)│
 └──────────┴──────────┴──────────┘
      ↓            ↓           ↓
 OMR 사이트     교재 점검    수학자 탐험관
  이동(외부)    (내장 기능)   (내장 기능)
```

### 각 메뉴의 구현 방식
| 메뉴 | 방식 | 이유 |
|---|---|---|
| OMR 답안 입력 | 외부 링크 이동 | 다른 강사 학생들도 사용, 독립 유지 필수 |
| 교재 점검 확인 | 포털 내 내장 | 수최쌤 담당반 핵심 기능 |
| 수학자 탐험관 | 포털 내 내장 (추후 개발) | 담당반 학습 콘텐츠, 개발 재개 시 통합 |

### 절대 바꾸지 않아야 할 것
- OMR 시스템의 기존 동작 방식 (다른 강사 학생들도 사용하므로)
- 교재 점검 시스템의 Firebase 데이터 구조
- 기존 강사용 입력 흐름
- 수학자 탐험관의 기존 데이터 (개발 중단 상태라도 데이터는 보존)

## 🛠 기술적 배경 (기존 교재 점검 프로젝트 기준)

- **프레임워크**: Vite + React
- **스타일**: Tailwind CDN (레거시), 신규 컴포넌트는 `src/styles/`
- **Firebase**: npm 패키지로 번들링, Firestore + Auth + Hosting 사용
- **레거시 코드**: `src/legacy/legacyApp.js` + `src/styles/legacy.css`
- **신규 React 위치**: `src/components`, `src/lib`, `src/styles`
- **배포**: Firebase Hosting, public 경로는 `dist/`
- **브라우저 기본 팝업 금지**: 커스텀 모달(`showModalAlert` 등) 필수 사용

### 코딩 스타일 원칙 (반드시 준수)
- 강사용 입력은 클릭 중심, 타이핑 최소화
- 학부모/학생 보고서는 시각화(막대그래프, 레이더차트) 우선
- 관리자 페이지는 카드형 아코디언, 최초 진입 시 접힌(collapsed) 상태
- 다크 테마 기준: `bg-slate-900`, `border-slate-800`, `text-slate-200`
- Apple 스타일: 여백, 타이포그래피, 미세한 hover 반응, 부드러운 Aurora 배경

## 📋 새 프로젝트에서 AI가 해야 할 첫 번째 작업

1. **세 폴더의 구조를 모두 파악**한다
   - 교재 점검 폴더: 기존 `AGENTS.md`, `RULES.md`, `memory/lessons.md`, `docs/` 참고
   - OMR 폴더: 폴더 구조 및 진입점 파악
   - 수학자 탐험관 폴더: 폴더 구조 및 현재 개발 상태 파악

2. **통합 방향 결정**을 위한 분석을 먼저 보고한다:
   - 세 프로젝트의 로그인/인증 방식이 동일한가?
   - 공통으로 쓸 수 있는 컴포넌트가 있는가?
   - Firebase 프로젝트가 각각 분리되어 있는가? (통합 필요 여부 판단)
   - OMR은 외부 링크로, 교재 점검과 수학자 탐험관은 내장으로 처리 가능한가?
   - 수학자 탐험관의 현재 개발 진행 상태 파악 (어디까지 만들었는가)

3. **구현 계획(implementation_plan.md)**을 먼저 작성하고 사용자 승인을 받은 뒤 코딩을 시작한다.

> ⚠️ 수학자 탐험관은 개발 중단 상태이므로, 포털 통합 시 메뉴는 표시하되 "준비 중" 상태로 두는 방식을 기본으로 제안한다. 사용자가 개발 재개를 원할 때 이어서 진행한다.

# AI Agent Guide

이 파일은 열린학원 수학교재점검 시스템을 함께 개발하는 AI 에이전트가 매 세션 시작 시 먼저 읽어야 하는 공동 작업 지침입니다.

## 프로젝트 정체성

- 서비스명: 열린학원 수학교재점검 시스템
- 내부명: OATIS, Open Academy Textbook Insight System
- 핵심 사용자: 학원 강사, 관리자, 학부모
- 핵심 목표: 강사는 교재 점검을 최대한 빠르게 입력하고, 학부모는 학생의 교재 관리 상태를 한눈에 이해한다.

## 반드시 먼저 읽을 파일

작업 전 다음 파일을 순서대로 확인한다.

1. `RULES.md`
2. `memory/lessons.md`
3. `docs/product-direction.md`
4. `docs/architecture.md`

작업 중 새로운 시행착오나 사용자의 선호가 확인되면 `memory/lessons.md`에 누적한다.

## 현재 기술 상태

- 현재 앱은 Vite/React 프로젝트로 전환되었다.
- 주요 기존 구현은 `src/legacy/legacyApp.js`와 `src/styles/legacy.css`로 분리되어 있다.
- 새 코드의 기본 위치는 `src/components`, `src/lib`, `src/styles`다.
- Firebase는 npm 패키지를 통해 Vite에서 번들링한다.
- 레거시 화면은 Tailwind CDN을 사용 중이다. 신규 React 컴포넌트는 필요한 스타일을 `src/styles`에 둔다.

## 작업 원칙

- 강사용 입력 화면은 클릭 중심이어야 한다. 반복 타이핑을 늘리는 변경은 피한다.
- 학부모 보고서는 시각적 요약을 우선한다. 긴 텍스트 설명보다 막대그래프, 비교표, 레이더 차트가 우선이다.
- 반별 보고서는 반 전체의 상태를 숫자로 비교할 수 있어야 한다.
- 관리자 페이지는 항상 섹션별 카드 형식으로 메뉴를 정리하며, 최초 진입 시에는 아코디언 방식으로 접힌(collapsed) 상태여야 한다.
- 모든 프로젝트 구성은 관리자 메뉴에서 설정 및 조정 시 실시간 반영되도록 진행한다.
- 기존 Firebase 컬렉션과 저장 데이터를 깨뜨리는 변경은 매우 신중하게 한다.
- 새 프로젝트나 새 목업에서 핵심 화면과 반복 UI가 보이기 시작하면, 기능을 계속 쌓기 전에 Vite/React 분리 시점을 먼저 판단한다.
- 분리 시점이라고 판단되면 사용자에게 먼저 제안하고, 승인 후 구조 분리와 검증을 완료한 다음 기능 개발을 이어간다.
- 레거시 코드를 옮길 때는 한 화면 또는 한 계산 단위씩 이동하고, 이동 전후 동작을 검증한다.
- 배포 전에는 로컬에서 화면이 깨지지 않는지 확인하고, 가능한 경우 Firebase Hosting 기준으로 검증한다.

## 개발 언어와 응답 방식

- 사용자와의 대화, 문서, 커밋 설명은 기본적으로 한국어로 작성한다.
- 코드 식별자와 Firebase 필드는 영어를 사용한다.
- 변경 사항은 작업 목적, 수정 파일, 검증 결과 중심으로 간결하게 보고한다.
- 세션의 작업이 어느 정도 묶음이 완료되었을 때는 사용자가 즉시 확인하고 반영할 수 있도록 큰 커밋 제목과 본문 내용을 복사 가능하게 정리하고, 실제 실행해야 할 배포 명령(예: firebase deploy --only hosting)은 별도의 코드블록으로 독립적으로 분리하여 제시한다.

## 금지 사항

- 사용자의 명시적 요청 없이 대규모 프레임워크 전환을 하지 않는다.
- 기존 데이터 구조를 마이그레이션 없이 바꾸지 않는다.
- 강사용 입력 흐름에 불필요한 텍스트 입력 단계를 추가하지 않는다.
- 보고서 출력 화면에서 모바일/인쇄 레이아웃을 확인하지 않고 완료했다고 말하지 않는다.

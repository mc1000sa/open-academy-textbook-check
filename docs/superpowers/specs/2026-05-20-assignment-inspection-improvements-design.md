# 과제 점검 시스템 개선 설계

## 목적

선생님이 교재 1권 단위로 과제를 점검할 때, 이번 회차 과제 수행률과 지난 회차 미완료 과제 회수 상태를 분리해서 확인하고 저장할 수 있게 한다. 학부모와 학생 보고서에서는 과제 수행률을 포함한 6요소를 학생 개인, 같은 교재 평균, 반 평균과 비교해 보여준다.

## 확정된 요구사항

- 점검 단위는 `반 + 날짜 + 교재 + 학생`이다.
- 한 학생이 교재를 2권 제출해도 교재 1권씩 별도로 점검한다.
- 과거 점검 기록은 수정하지 않고 원본 이력으로 보존한다.
- 지난 미완료 과제를 이번 회차에서 완료하면, 이번 회차 점검 기록에 회수 이력을 남긴다.
- 지난 미완료 과제는 같은 학생, 같은 교재의 과거 기록에서 가져온다.
- 선생님 화면에는 지난 점검일과 미완료 페이지가 표 형태로 표시된다.
- 페이지 버튼은 기본 미선택 상태이며, 누르면 이번 회차 완료로 선택된다. 다시 누르면 선택 해제된다.
- 저장 시 이번 회차 과제 점검, 지난 미완료 완료 선택, 5요소 점수를 함께 저장한다.
- 이번 회차 과제 수행률과 지난 미완료 회수율은 합산하지 않고 별도 지표로 표시한다.

## 6요소 명칭

육각형 비교에 사용하는 6요소는 다음 이름을 사용한다.

| 요소 | 입력 방식 |
| --- | --- |
| 과제 수행률 | 교재 범위와 이번 회차 미완료 페이지로 자동 계산 |
| 풀이 표현력 | 0~10점 버튼 입력 |
| 채점 성실도 | 0~10점 버튼 입력 |
| 수업 태도 | 0~10점 버튼 입력 |
| 개념 이해도 | 0~10점 버튼 입력 |
| 응용 해결력 | 0~10점 버튼 입력 |

5개 정성 요소는 0.5점 단위 입력을 지원한다. 기본 UI는 정수 점수 버튼과 0.5 보정 컨트롤을 함께 제공해, 21개 버튼이 한 줄에 과도하게 늘어나는 문제를 피한다.

## 점검 화면 설계

선생님이 반, 날짜, 교재를 선택하면 해당 교재를 점검할 학생 목록과 각 학생의 과거 미완료 표를 보여준다. 과거 미완료 표는 날짜별 행으로 구성한다.

| 지난 점검일 | 지난 회차 미완료 페이지 | 이번 회차 완료 선택 |
| --- | --- | --- |
| 2026-05-10 | p.12, p.13, p.15 | 페이지별 선택 버튼 |
| 2026-05-13 | p.22, p.24 | 페이지별 선택 버튼 |

페이지 버튼 상태는 다음과 같다.

- 미선택: 아직 미완료 유지
- 선택: 이번 회차에 완료 확인
- 재클릭: 선택 취소

저장 전에는 선택된 페이지가 색상으로 명확히 구분되어야 한다. 저장 버튼은 이번 회차 점검 저장 버튼 하나로 통합해 선생님이 여러 저장 버튼 사이에서 혼동하지 않게 한다.

## 데이터 저장 설계

기존 Firestore 컬렉션과 기존 필드 의미는 유지한다. 새 필드는 점검 기록 문서에 하위 호환 방식으로 추가한다.

예상 필드:

```js
{
  completionRate: 85,
  rangeStart: 10,
  rangeEnd: 30,
  missedPages: [12, 18],
  carryoverResolutions: [
    {
      sourceInspectionId: "inspection-id",
      sourceDate: "2026-05-10",
      resolvedPages: [13, 15]
    }
  ],
  qualityScores: {
    expression: 8.5,
    grading: 9,
    attitude: 8,
    understanding: 7.5,
    application: 8
  }
}
```

필드명은 구현 전 실제 코드의 기존 naming pattern을 확인한 뒤 확정한다. 오래된 점검 기록에는 새 필드가 없을 수 있으므로 계산 함수는 누락값을 안전하게 처리한다.

## 계산 기준

이번 회차 과제 수행률은 기존 계산 기준을 유지한다.

- 이번 회차 과제 수행률: `rangeStart`, `rangeEnd`, `missedPages` 기반 자동 계산
- 육각형 과제 수행률 점수: `completionRate / 10`으로 10점 스케일 변환
- 지난 미완료 회수율: 과거 미완료 페이지 수 대비 이번 회차에서 완료 선택한 페이지 수

이번 회차 과제 수행률과 지난 미완료 회수율은 서로 다른 의미의 지표이므로 하나의 점수로 합산하지 않는다.

## 보고서 설계

학생·학부모 보고서에는 개인 육각형과 평균 육각형을 비교해서 표시한다.

- 학생 개인 육각형: 해당 학생의 누적 6요소 평균
- 같은 교재 평균 육각형: 학원 전체 기록 중 같은 교재를 점검한 모든 학생의 평균
- 반 평균 육각형: 해당 반 전체 학생의 평균

보고서에서는 다음 상태를 구분해서 보여준다.

- 이번 회차 과제 수행률
- 지난 미완료 회수율
- 지난 미완료 중 이번 회차 완료로 확인된 페이지
- 아직 남아 있는 미완료 페이지

인쇄와 PDF 저장을 고려해 `@media print`에서 표와 육각형 비교가 깨지지 않아야 한다.

## 구현 방향

추천 방식은 단계적 기능 추가다.

1. 기존 점검 기록 구조를 유지하고 새 필드를 하위 호환으로 추가한다.
2. 과거 미완료 조회와 회수율 계산은 `src/lib`의 순수 함수로 분리한다.
3. 선생님 점검 화면에 날짜별 과거 미완료 표와 페이지 선택 UI를 추가한다.
4. 5요소 점수 입력 UI를 버튼형으로 추가한다.
5. 보고서 계산 함수에 개인, 같은 교재 평균, 반 평균 6요소 집계를 추가한다.
6. 보고서 화면에 육각형 비교 시각화를 추가한다.

## 검증 기준

- 오래된 점검 기록에 새 필드가 없어도 화면과 보고서가 깨지지 않는다.
- 과거 미완료 기록은 저장 후에도 원본이 변경되지 않는다.
- 이번 회차 기록에 지난 미완료 회수 이력이 남는다.
- 같은 학생, 같은 교재의 과거 미완료만 표에 표시된다.
- 이번 회차 과제 수행률과 지난 미완료 회수율이 별도 지표로 표시된다.
- 5요소 점수는 버튼으로 빠르게 입력할 수 있다.
- 보고서에서 학생 개인, 같은 교재 평균, 반 평균 육각형 비교가 가능하다.
- `npm.cmd test`, `npm.cmd run build`, 주요 화면 브라우저 확인을 통과한다.
